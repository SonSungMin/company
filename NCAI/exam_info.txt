# NCAI (Naver Cloud AI) 자격증 합격 완벽 가이드

## 📌 시험 기본 정보

### 시험 개요
NCAI는 네이버 클라우드의 인공지능 서비스 전반을 파악하고 인공지능의 이해를 요구하는 자격증입니다. 다른 네이버 클라우드 자격과 달리 인공지능 알고리즘, 인공지능 개념 등에 대한 이해도가 높이 필요합니다.

### 시험 특징
- **1과목 구성**: 다른 NCP 자격증과 달리 1개 과목으로 구성
- **응시 방식**: 오프라인으로만 응시가 가능
- **시험 난이도**: NCE 수준보다 더 상세하고 정확한 내용 요구, 다항 문제가 많음
- **학습 범위**: 인공지능 기초 + AI 서비스 + API 사용법 및 파라미터

## 🎯 핵심 학습 영역

### 1. 인공지능 기초 이론 (필수)

NCAI는 다른 네이버 클라우드 자격증과 달리 **AI 이론**을 깊이 있게 다룹니다.

#### 1.1 머신러닝 기초

**학습의 종류**
- 지도학습: 문제와 정답을 알려주고 학습
- **비지도학습**: 정답 없이 데이터의 패턴 발견 (클러스터링, 차원 축소)
- **강화학습**: 보상을 통한 학습

**주요 알고리즘**
1. **Decision Tree (결정 트리)**
   - 트리 구조로 의사결정 규칙을 학습
   - 해석이 쉽지만 과적합 위험
   - Gini impurity, Entropy로 분할 기준 결정

2. **Random Forest**
   - 결정 트리를 여러 개 엮어 과적합 문제를 해소
   - 앙상블 기법 (Bagging)
   - 개별 트리보다 일반화 성능 우수

3. **SVM (Support Vector Machine)**
   - 초평면을 이용한 분류
   - 커널 트릭 활용

4. **K-Means Clustering**
   - 비지도학습의 대표 알고리즘
   - 중심점 기반 군집화

5. **Neural Networks & Deep Learning**
   - 활성화 함수 (ReLU, Sigmoid, Tanh)
   - 역전파 알고리즘
   - CNN, RNN, Transformer

#### 1.2 머신러닝 핵심 개념

**과적합 vs 일반화**
- 과적합(Overfitting): 학습 데이터에만 과도하게 맞춤
- **일반화(Generalization)**: 새로운 데이터에 대한 성능
- **정규화(Regularization)**: L1, L2 정규화로 과적합 방지

**데이터셋 분할**
- Training Set, Validation Set, Test Set으로 분할
- **교차 검증(Cross Validation)**: K-Fold CV

**평가 지표**
- **분류**: Accuracy, Precision, Recall, F1-Score, AUC-ROC
- **회귀**: MSE, RMSE, MAE, R²

**하이퍼파라미터 vs 파라미터**
- **파라미터**: 학습을 통해 자동으로 결정 (가중치, 편향)
- **하이퍼파라미터**: 학습 전에 설정 (learning rate, batch size, epoch)

### 2. 네이버 클라우드 AI 서비스 (상세)

네이버 클라우드 상품에 대해 NCE보다 더 자세한 사항을 요구합니다.

#### 2.1 CLOVA Speech Recognition (CSR) - 음성 인식

**핵심 특징**
- 한국어, 영어, 일어, 중국어(간체) 지원
- 머신러닝 방법이 적용되어 서비스 성능이 지속적으로 개선
- 모바일 SDK와 REST API 방식 모두 제공

**API 사용 세부사항**
- API 호출 1회당 인식할 수 있는 음성은 60초까지 가능
- 이용 한도: Application 당 월 300,000초(일 10,000초) ~ 월 30,000,000초
- 이용량은 15초 단위로 올림하여 계산

**주요 파라미터**
- `language`: 인식할 언어 (Kor, Jpn, Eng, Chn)
- `completion`: 완료 여부 (완료/진행중)
- Audio format: PCM, MP3, WAV

#### 2.2 CLOVA Voice - 음성 합성 (TTS)

**핵심 특징**
- 총 100가지 자연스러운 음성 합성음 제공
- Pro 버전은 NeuVis(Neural Voice Synthesis) 엔진 이용
- 1회 호출 시 최대 2,000글자 요청 가능

**감정 파라미터**
- Volume, Speed, Pitch, Emotion 등의 감정 파라미터 제공
- 출력 형식: *.mp3, *.wav

**사용 제한**
- 실시간 API 호출로 AI 보이스를 생성하며, 생성된 파일을 저장하거나 편집하여 재사용 불가

#### 2.3 CLOVA Chatbot

**다국어 지원**
- 지원 언어: 한국어, 일본어, 영어, 중국어, 태국어, 인도네시아어
- 각 언어별 특징에 맞는 최적화된 학습 알고리즘

**주요 기능**
- 자연어 처리 기술과 머신러닝 기반 학습 알고리즘
- 답변 컴포지트: Text, 버튼, 다중 버튼, 이미지, 캐로셀

**도메인 구조**
- Domain / Domain Group 선택 가능
- Domain Group의 API 호출은 Transaction 기준 2회 호출로 카운트

#### 2.4 CLOVA Studio - 생성형 AI 플랫폼

**HyperCLOVA X**
- 한국어 특화 토크나이저 활용, 해외 LLM 대비 최고 2배까지 빠르고 저렴
- 이미지와 음성 정보를 한국어로 추론 가능한 네이티브 멀티모달
- 한국의 문화, 역사, 언어, 사회 규범에 대한 이해와 추론 뛰어남

**서비스 유형**
- CLOVA Studio Basic: 토큰 사용량에 따라 요금 지불, 신규 기능 빠른 적용
- **CLOVA Studio Exclusive**: Advanced Tuning 전용 인프라 제공
- **Neurocloud for HyperCLOVA X**: 고객사 IDC에 하이브리드 클라우드 형태로 제공하는 완전관리형 서비스

**API 키 관리**
- API 키가 제3자에게 유출되면 보안 문제 발생 가능
- 삭제 및 재발급 가능

#### 2.5 CLOVA OCR

**서비스 유형**
- **일반 OCR**: 다양한 형태의 텍스트 인식
- **템플릿 OCR**: 특정 양식(신분증, 명함, 영수증) 인식
- **Custom OCR**: 사용자 정의 양식 학습

**핵심 기능**
- 이미지에서 텍스트 추출
- 좌표 정보 제공
- 정확도 점수 제공

#### 2.6 Papago Translation

**번역 서비스**
- 다국어 번역 지원
- 텍스트, 웹사이트, 이미지 번역
- Neural Machine Translation (NMT) 기반

#### 2.7 기타 AI 서비스

**CLOVA Face Recognition (CFR)**
- 얼굴 감지 및 특징점 추출
- 유명인 얼굴 인식
- 얼굴 비교

**CLOVA Pose Estimation**
- 신체 주요 관절 위치 추출
- 스켈레톤 정보 제공

**Object Detection**
- 이미지 내 객체 탐지 및 분류
- 바운딩 박스 좌표

**Sentiment Analysis**
- 텍스트의 긍정/부정/중립 판단
- 감정 점수 제공

### 3. 딥러닝 & NLP 개념

#### 3.1 자연어 처리 (NLP)

**토크나이제이션**
- 문장을 토큰(단어, 형태소)으로 분리
- 한국어 특화 토크나이저의 중요성

**임베딩**
- Word2Vec, GloVe, FastText
- 문맥 기반 임베딩: BERT, GPT

**주요 NLP 작업**
- 텍스트 분류
- 개체명 인식 (NER)
- 질의응답 (QA)
- 요약
- 번역

#### 3.2 Transformer & Attention

**Attention Mechanism**
- Self-Attention
- Multi-Head Attention
- 문맥 이해 능력 향상

**Transformer 구조**
- Encoder-Decoder 구조
- Position Encoding
- Layer Normalization

**LLM (Large Language Model)**
- GPT, BERT, T5
- HyperCLOVA X: 한국의 문화와 맥락을 가장 잘 이해하는 생성형 AI

#### 3.3 Computer Vision

**CNN (Convolutional Neural Network)**
- Convolution Layer
- Pooling Layer
- 특징 추출

**주요 아키텍처**
- LeNet, AlexNet, VGG
- ResNet (Residual Connection)
- Vision Transformer (ViT)

### 실제 합격자의 조언

시험 당시 NCE 수준으로 파라미터 등을 물어볼 것이라 생각했는데, 그것보다 상세하고 정확한 내용을 요구해서 조금 당황했습니다.
