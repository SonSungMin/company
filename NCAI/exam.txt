## 섹션 1: CLOVA Studio 개요 및 기초

**1. 네이버의 초대규모(Hyperscale) AI 모델인 'HyperCLOVA X'를 활용하여, 코딩 없이도 AI 모델을 테스트하고 서비스를 만들 수 있는 개발 플랫폼의 명칭은?**
① CLOVA OCR
② CLOVA Studio
③ CLOVA Dubbing
④ CLOVA Vision

**정답: ②**
해설: CLOVA Studio는 HyperCLOVA X를 기반으로 프롬프트 엔지니어링, 튜닝 등을 수행할 수 있는 노코드(No-Code) AI 개발 도구입니다.

**2. CLOVA Studio에서 사용자가 원하는 결과값을 얻기 위해 모델에 입력하는 명령어 또는 예시문을 의미하는 용어는?**
① 데이터셋(Dataset)
② 파라미터(Parameter)
③ 프롬프트(Prompt)
④ 토큰(Token)

**정답: ③**
해설: 프롬프트는 AI 모델이 특정 작업을 수행하도록 유도하는 입력 텍스트를 의미합니다.

**3. CLOVA Studio의 '플레이그라운드(Playground)' 모드 중 하나로, 대화형 인터페이스를 통해 모델과 상호작용하며 챗봇 서비스를 설계하기에 가장 적합한 모드는?**
① Completion
② Chat
③ Skill
④ Tuning

**정답: ②**
해설: Chat 모드는 멀티 턴(Multi-turn) 대화를 테스트하고 설계하는 데 최적화되어 있습니다.

**4. 다음 중 프롬프트의 품질을 높이기 위한 '프롬프트 엔지니어링' 기법으로 가장 적절하지 않은 것은?**
① 명확하고 구체적인 지시문을 작성한다.
② 모델이 참고할 수 있는 입출력 예시(Few-shot)를 제공한다.
③ 한꺼번에 너무 많은 복합적인 임무를 한 문장에 넣는다.
④ 답변의 형식을 지정하여 출력 결과의 일관성을 높인다.

**정답: ③**
해설: 복잡한 임무는 단계를 나누어 명확히 지시하는 것이 모델의 정확도를 높이는 데 유리합니다.

**5. [다지선다] CLOVA Studio에서 HyperCLOVA X 모델의 답변 성향을 조절하기 위해 설정할 수 있는 파라미터를 모두 고르시오.**
① Temperature (온도)
② Top P (Nucleus Sampling)
③ Max Tokens
④ Stop Sequences

**정답: ①,②,③,④**
해설: 네 가지 모두 답변의 창의성, 확률적 선택 범위, 길이, 중단 지점 등을 제어하는 주요 파라미터입니다.

**6. 파라미터 중 'Temperature'에 대한 설명으로 옳은 것은?**
① 값이 낮을수록 모델이 더 다양하고 창의적인 답변을 생성한다.
② 값이 높을수록 모델이 더 확률적으로 높은, 일관된 답변을 생성한다.
③ 0에 가까울수록 결정론적(Deterministic)이고 보수적인 답변을 생성한다.
④ 답변의 최대 길이를 제한하는 데 사용된다.

**정답: ③**
해설: Temperature 값이 낮을수록 모델은 가장 확률이 높은 단어를 선택하여 일관성 있는 답변을 제공합니다.

**7. 특정 도메인의 데이터를 학습시켜 HyperCLOVA X 모델을 사용자 목적에 맞게 최적화하는 과정을 무엇이라 하는가?**
① 인코딩(Encoding)
② 튜닝(Tuning)
③ 임베딩(Embedding)
④ 스케일링(Scaling)

**정답: ②**
해설: 튜닝(Tuning)은 사전 학습된 모델을 특정 데이터셋으로 재학습시켜 성능을 고도화하는 과정입니다.

**8. CLOVA Studio에서 튜닝을 진행하기 위해 준비해야 하는 데이터셋 파일의 형식은?**
① CSV
② JSONL
③ XLSX
④ TXT

**정답: ②**
해설: CLOVA Studio의 튜닝 데이터셋은 한 줄씩 JSON 객체로 구성된 JSONL 형식을 사용해야 합니다.

**9. [다지선다] CLOVA Studio의 'Skill' 기능에 대한 설명으로 옳은 것을 모두 고르시오.**
① 외부 서비스의 API를 호출하여 답변에 활용할 수 있게 한다.
② 실시간 정보(날씨, 뉴스 등)를 반영한 답변이 가능해진다.
③ 튜닝 없이도 모델의 지식을 최신화하는 효과를 낼 수 있다.
④ 오직 네이버 내부 API만 연동 가능하다.

**정답: ①,②,③**
해설: Skill은 외부 도구나 API를 모델과 결합하여 모델 지식의 한계를 극복하게 해주는 기능입니다.

**10. CLOVA Studio에서 생성한 프롬프트를 실제 서비스에 적용하기 위해 발급받아야 하는 인증 정보는?**
① API Key
② 인증서(Certificate)
③ ID/PW
④ 보안 토큰

**정답: ①**
해설: 생성된 테스트 앱을 외부에서 API로 호출하기 위해서는 API Key 발급이 필수적입니다.

**11. CLOVA Studio의 'Explorer' 메뉴가 제공하는 주요 기능은?**
① 모델의 코드를 직접 수정하는 기능
② 다양한 프롬프트 사례와 템플릿을 확인하고 바로 사용해 볼 수 있는 기능
③ 결제 수단을 변경하는 기능
④ 데이터베이스를 관리하는 기능

**정답: ②**
해설: Explorer는 다른 사용자가 공유하거나 네이버에서 제공하는 우수한 프롬프트 예시 모음집입니다.

**12. 답변의 창의성을 제어하는 'Top P' 파라미터에 대한 설명으로 옳은 것은?**
① 누적 확률이 P인 범위 내의 후보 단어군 중에서만 선택한다.
② 답변의 전체 길이를 P 토큰으로 제한한다.
③ 답변의 속도를 P배 만큼 빠르게 한다.
④ 특정 단어 P가 나오지 않도록 차단한다.

**정답: ①**
해설: Top P(Nucleus Sampling)는 상위 확률 단어들의 누적 합을 기준으로 후보군을 필터링하는 방식입니다.

**13. [다지선다] 튜닝(Tuning) 과정 중 '데이터셋 준비' 단계에서 주의해야 할 사항을 모두 고르시오.**
① 데이터의 양보다는 질(Quality)이 중요하다.
② 학습 데이터에는 부적절하거나 편향된 내용이 포함되지 않아야 한다.
③ 프롬프트와 답변의 쌍이 의도에 맞게 정확히 구성되어야 한다.
④ 데이터셋의 크기가 크면 검증 과정은 생략해도 된다.

**정답: ①,②,③**
해설: 튜닝 데이터는 정확도와 정제도가 핵심이며, 반드시 검증(Validation) 과정을 거쳐야 합니다.

**14. CLOVA Studio 이용 시 발생하는 비용 산정의 가장 핵심적인 기준은?**
① 접속 시간(Minute)
② 처리된 토큰 수(Token Count)
③ 생성된 프로젝트 수
④ 업로드한 파일 크기(MB)

**정답: ②**
해설: HyperCLOVA X와 같은 LLM 서비스는 입/출력에 사용된 토큰 수를 기반으로 과금됩니다.

**15. HyperCLOVA X의 '안전한 AI(Safety)'를 위해 CLOVA Studio에서 기본 제공하는 기능은?**
① 자동 번역 기능
② 유해 콘텐츠 필터링(Content Filtering)
③ 오타 교정 기능
④ 이미지 생성 기능

**정답: ②**
해설: 혐오 표현이나 위험한 답변을 사전에 차단하기 위한 필터링 시스템이 내장되어 있습니다.

**16. [다지선다] CLOVA Studio에서 지원하는 '시스템 메시지(System Message)'의 역할로 옳은 것을 모두 고르시오.**
① AI의 역할(Role)을 정의한다.
② 답변의 말투(Tone & Manner)를 지정한다.
③ 대화의 규칙이나 제약사항을 설정한다.
④ 사용자가 입력하는 구체적인 질문 그 자체이다.

**정답: ①,②,③**
해설: 시스템 메시지는 모델에게 지침을 내리는 배경 설정 정보이며, 사용자의 질문은 'User' 메시지에 해당합니다.

**17. HyperCLOVA X 모델이 가진 지식을 넘어, 사내 문서 등을 검색하여 답변하도록 하는 RAG 구성 시 필수적인 API는?**
① 텍스트 생성 API
② 임베딩(Embedding) API
③ 이미지 분석 API
④ 음성 인식 API

**정답: ②**
해설: 문서를 벡터화하여 검색하기 위해서는 텍스트를 숫자로 변환하는 임베딩 API가 반드시 필요합니다.

**18. 튜닝 작업의 상태 중, 학습이 정상적으로 완료되어 모델 배포가 가능한 상태는?**
① Queued
② Running
③ Succeeded
④ Failed

**정답: ③**
해설: 튜닝 진행 결과가 Succeeded로 표시되어야 배포 및 서비스 적용이 가능합니다.

**19. [다지선다] 프롬프트 작성 시 'Few-shot' 방식이 'Zero-shot' 방식보다 효과적인 경우를 모두 고르시오.**
① 출력 형식이 JSON이나 특정 리스트로 엄격해야 할 때
② 모델에게 복잡한 논리적 추론 과정을 보여주어야 할 때
③ 단순한 단어의 의미를 물어볼 때
④ 일관된 답변 스타일을 유지해야 할 때

**정답: ①,②,④**
해설: 예시(Shot)를 제공하면 모델은 패턴을 더 잘 이해하여 정확한 형식과 논리로 답변합니다.

**20. CLOVA Studio에서 테스트한 프롬프트를 서비스에 적용하기 위해 '테스트 앱'을 생성했을 때 제공되지 않는 정보는?**
① API Key
② API Endpoint URL
③ Client Secret
④ 사용자 개인 비밀번호

**정답: ④**
해설: API 호출에 필요한 인증 Key와 Secret, URL 정보는 제공되지만 개인 비밀번호는 포함되지 않습니다.

**21. 튜닝 학습 과정에서 'Epoch'이 의미하는 수치는?**
① 학습 데이터셋 전체를 반복 학습하는 횟수
② 한 번에 처리하는 데이터의 크기
③ 학습에 소요되는 총 시간
④ 학습 시 사용되는 단어의 총합

**정답: ①**
해설: Epoch은 전체 데이터셋을 모델이 몇 번 반복하여 학습할지를 결정하는 단위입니다.

**22. [다지선다] Chat 플레이그라운드에서 메시지의 역할을 구분하는 요소를 모두 고르시오.**
① System (시스템 설정)
② User (사용자 질문)
③ Assistant (AI 답변)
④ Guest (비회원 질문)

**정답: ①,②,③**
해설: CLOVA Studio는 System, User, Assistant의 세 가지 역할을 통해 대화 문맥을 관리합니다.

**23. 프롬프트 내에서 지시 사항과 데이터를 명확히 분리하기 위해 사용하는 '###'이나 '---' 같은 기호의 명칭은?**
① 구분자(Delimiter)
② 연산자(Operator)
③ 주석(Comment)
④ 토큰(Token)

**정답: ①**
해설: 구분자를 사용하면 모델이 프롬프트의 구조를 더 잘 파악하여 정확도가 올라갑니다.

**24. [다지선다] CLOVA Studio에서 모델을 선택할 때 고려해야 할 사항으로 옳은 것을 모두 고르시오.**
① 작업의 복잡도 (높을수록 큰 모델 권장)
② 응답 속도 (빠를수록 경량 모델 권장)
③ 서비스 비용 (경량 모델이 저렴함)
④ 데이터 보안 (모든 모델이 동일한 보안 환경 적용)

**정답: ①,②,③**
해설: 성능 중심인지, 비용/속도 중심인지에 따라 적절한 모델 크기를 선택해야 합니다.

**25. 튜닝 실행 전, 데이터셋 파일의 문법 오류나 형식 누락을 확인하는 과정을 무엇이라 하는가?**
① 데이터셋 검증(Validation)
② 데이터셋 삭제
③ 데이터셋 공유
④ 데이터셋 암호화

**정답: ①**
해설: 데이터셋 업로드 시 검증 단계를 거쳐야 학습 오류를 방지할 수 있습니다.

**26. HyperCLOVA X가 한국어 처리에 강점을 가지는 근본적인 이유는?**
① 영어 데이터를 한국어로 자동 번역해서 학습했기 때문
② 한국어와 한국 문화 관련 데이터를 대규모로 학습했기 때문
③ 일본에서 개발된 모델이기 때문
④ 오직 한국어 데이터만 학습했기 때문

**정답: ②**
해설: HyperCLOVA X는 방대한 양의 한국어 및 사회/문화적 맥락 데이터를 학습한 모델입니다.

**27. Playground 설정 중, 특정 문자가 나오면 답변 생성을 즉시 멈추도록 하는 기능은?**
① Max Tokens
② Stop Sequences
③ Temperature
④ Top P

**정답: ②**
해설: Stop Sequences에 중단 문자를 설정하면 해당 문자 생성 시 답변이 종료됩니다.

**28. [다지선다] CLOVA Studio 프로젝트 관리 기능에서 멤버 초대 시 부여할 수 있는 권한 종류를 모두 고르시오.**
① 마스터 (Master)
② 매니저 (Manager)
③ 뷰어 (Viewer)
④ 게스트 (Guest)

**정답: ①,②,③**
해설: 프로젝트별로 마스터, 매니저, 뷰어 권한을 부여하여 협업할 수 있습니다.

**29. 모델이 이전 대화의 흐름을 기억하지 못할 때(Stateless), 이를 해결하기 위해 프롬프트에 포함해야 하는 정보는?**
① API Key 이력
② 지금까지의 대화 이력 (History)
③ 사용자의 결제 내역
④ 튜닝 로그 파일

**정답: ②**
해설: LLM은 기본적으로 상태를 저장하지 않으므로, 이전 대화 문맥을 다시 전달해 주어야 연속된 대화가 가능합니다.

**30. CLOVA Studio 'Playground' 우측 상단의 '코드 보기'를 통해 얻을 수 있는 정보는?**
① 모델의 딥러닝 아키텍처 소스 코드
② 현재 설정을 API로 호출하기 위한 샘플 코드
③ 네이버 클라우드의 전체 서버 설정값
④ 실시간 과금 상세 내역

**문제 1. 인공 신경망(Neural Network)의 발전 과정과 역사에 대한 설명으로 옳은 것은?**
① 퍼셉트론은 다층 구조를 통해 XOR 문제를 완벽히 해결하며 등장했다.
② RNN(Recurrent Neural Network)은 데이터의 순차적 정보(Sequence)를 처리하기에 적합한 구조이다.
③ 트랜스포머(Transformer) 모델은 RNN의 병렬 처리 한계를 극복하기 위해 은닉 상태(Hidden State)를 강화한 모델이다.
④ 초기 AI 역사는 딥러닝 -> 머신러닝 -> 인공지능 순으로 포괄적인 개념이 정립되었다.

**정답: ②**
해설: 퍼셉트론은 초기 XOR 문제를 해결하지 못해 AI의 겨울을 가져왔으며, 트랜스포머는 RNN의 순차적 처리 한계를 극복하기 위해 '어텐션(Attention)' 메커니즘을 도입하여 병렬화를 실현했습니다. AI의 포괄 범위는 인공지능 > 머신러닝 > 딥러닝 순입니다.

**문제 2. HyperCLOVA X의 튜닝(Tuning) 방식 중 'LoRA(Low-Rank Adaptation)'에 대한 설명으로 옳은 것은?**
① 모델의 전체 파라미터를 모두 업데이트하여 가장 높은 정확도를 보장한다.
② 학습 데이터의 양이 적을 때 주로 사용하며, 프롬프트 끝에 예시를 추가하는 방식이다.
③ 사전 학습된 모델의 가중치는 고정하고, 일부 저차원 행렬만을 학습시켜 자원 효율성을 높인다.
④ 별도의 학습 과정 없이 시스템 프롬프트(System Prompt) 수정만으로 성능을 개선한다.

**정답: ③**
해설: LoRA는 파라미터 효율적 미세 조정(PEFT) 기법 중 하나로, 거대 모델의 전체를 수정하지 않고 일부 가중치만 학습시켜 시간과 비용을 획기적으로 줄이는 튜닝 방법론입니다.
