📘 Part 1: CLOVA Studio - 생성형 AI의 중심
Chapter 1. CLOVA Studio 기본 개념
네이버의 초대규모 AI 모델인 HyperCLOVA X를 활용하여 코딩 없이도 AI 모델을 테스트하고 서비스를 만들 수 있는 개발 플랫폼이 바로 CLOVA Studio입니다. 이는 프롬프트 엔지니어링, 튜닝 등을 수행할 수 있는 노코드(No-Code) AI 개발 도구로, 개발자가 아니더라도 직관적인 인터페이스를 통해 강력한 AI 서비스를 구축할 수 있습니다.
CLOVA Studio에서 사용자가 원하는 결과를 얻기 위해 모델에 입력하는 명령어나 예시문을 **프롬프트(Prompt)**라고 합니다. 프롬프트는 AI 모델이 특정 작업을 수행하도록 유도하는 입력 텍스트를 의미하며, 이를 얼마나 잘 작성하느냐에 따라 AI의 성능이 크게 달라집니다.
Chapter 2. Playground 모드의 이해
CLOVA Studio의 플레이그라운드는 여러 모드로 구성되어 있습니다. 그중 Chat 모드는 멀티 턴(Multi-turn) 대화를 테스트하고 설계하는 데 최적화되어 있어, 대화형 인터페이스를 통해 모델과 상호작용하며 챗봇 서비스를 설계할 수 있습니다. 반면 Completion 모드는 단일 입력에 대한 출력을 생성하는 데 적합합니다.
Chat 모드에서는 메시지의 역할을 구분하는 세 가지 요소가 있습니다. **System(시스템 설정)**은 AI의 역할과 행동 방식을 정의하고, **User(사용자 질문)**는 사용자의 입력을, **Assistant(AI 답변)**는 모델의 응답을 나타냅니다. 이 구조를 통해 대화의 문맥을 효과적으로 관리할 수 있습니다.
Chapter 3. 프롬프트 엔지니어링의 기술
프롬프트의 품질을 높이기 위한 프롬프트 엔지니어링은 CLOVA Studio 활용의 핵심입니다. 좋은 프롬프트는 명확하고 구체적인 지시문을 포함하며, 모델이 참고할 수 있는 입출력 예시(Few-shot)를 제공합니다. 또한 답변의 형식을 지정하여 출력 결과의 일관성을 높일 수 있습니다.
반대로 피해야 할 것도 있습니다. 한꺼번에 너무 많은 복합적인 임무를 한 문장에 넣으면 모델이 혼란스러워할 수 있습니다. 복잡한 임무는 단계를 나누어 명확히 지시하는 것이 모델의 정확도를 높이는 데 유리합니다.
프롬프트 내에서 지시 사항과 데이터를 명확히 분리하기 위해 '###'이나 '---' 같은 **구분자(Delimiter)**를 사용하면 모델이 프롬프트의 구조를 더 잘 파악하여 정확도가 올라갑니다.
Few-shot 방식은 Zero-shot 방식보다 효과적인 경우가 많습니다. 특히 출력 형식이 JSON이나 특정 리스트로 엄격해야 할 때, 모델에게 복잡한 논리적 추론 과정을 보여주어야 할 때, 일관된 답변 스타일을 유지해야 할 때 예시를 제공하면 모델은 패턴을 더 잘 이해하여 정확한 형식과 논리로 답변합니다.
Chapter 4. HyperCLOVA X 파라미터의 이해
HyperCLOVA X 모델의 답변 성향을 조절하기 위해 설정할 수 있는 파라미터들이 있습니다. Temperature(온도), Top P(Nucleus Sampling), Max Tokens, Stop Sequences 등이 대표적입니다.
Temperature는 답변의 창의성을 제어하는 핵심 파라미터입니다. 값이 낮을수록 모델은 가장 확률이 높은 단어를 선택하여 일관성 있는 답변을 제공합니다. 0에 가까울수록 결정론적(Deterministic)이고 보수적인 답변을 생성하며, 값이 높을수록 모델이 더 다양하고 창의적인 답변을 생성합니다.
Top P는 누적 확률이 P인 범위 내의 후보 단어군 중에서만 선택하는 방식입니다. 예를 들어 Top P가 0.9라면 상위 확률 단어들의 누적 합이 90%가 될 때까지의 후보군만 고려합니다. 이는 답변의 다양성과 품질의 균형을 맞추는 데 유용합니다.
Repetition Penalty는 동일한 토큰이 반복될 확률을 낮춰 문장이 무의미하게 반복되는 것을 방지합니다. Presence Penalty는 단어의 반복 출현을 억제하여 더 다양한 표현을 사용하도록 유도합니다.
시나리오 작가처럼 창의적이고 매번 다른 답변을 생성해야 하는 작업에는 Temperature 값을 높이고(0.8 이상), Top P 값도 높이며(0.9 이상), Presence Penalty 값을 높여 중복 표현을 피하는 것이 좋습니다.
Chapter 5. 튜닝(Tuning)의 세계
특정 도메인의 데이터를 학습시켜 HyperCLOVA X 모델을 사용자 목적에 맞게 최적화하는 과정을 **튜닝(Tuning)**이라고 합니다. 이는 사전 학습된 모델을 특정 데이터셋으로 재학습시켜 성능을 고도화하는 과정입니다.
CLOVA Studio에서 튜닝을 진행하기 위해서는 JSONL 형식의 데이터셋 파일을 준비해야 합니다. 한 줄씩 JSON 객체로 구성된 이 형식은 일반적으로 **text(입력)**와 completion(출력) 컬럼으로 구성됩니다.
튜닝 작업의 상태 중 학습이 정상적으로 완료되어 모델 배포가 가능한 상태는 Succeeded입니다. Queued는 대기 중, Running은 학습 중, Failed는 실패를 의미합니다.
Epoch는 전체 데이터셋을 모델이 몇 번 반복하여 학습할지를 결정하는 단위입니다. 적절한 Epoch 수를 설정하는 것이 과적합을 방지하면서도 충분한 학습을 보장하는 핵심입니다.
튜닝 과정 중 '데이터셋 준비' 단계에서는 데이터의 양보다 **질(Quality)**이 중요합니다. 학습 데이터에는 부적절하거나 편향된 내용이 포함되지 않아야 하며, 프롬프트와 답변의 쌍이 의도에 맞게 정확히 구성되어야 합니다. 또한 반드시 검증(Validation) 과정을 거쳐야 합니다.
**LoRA(Low-Rank Adaptation)**는 모델의 전체 파라미터를 모두 업데이트하지 않고, 일부 파라미터만을 추가하여 효율적으로 학습시키는 방식입니다. 사전 학습된 모델의 가중치는 고정하고 일부 저차원 행렬만을 학습시켜 메모리와 비용을 절감하면서도 효과적으로 튜닝하는 기법입니다.
Chapter 6. CLOVA Studio 비용과 보안
CLOVA Studio 이용 시 발생하는 비용 산정의 가장 핵심적인 기준은 **처리된 토큰 수(Token Count)**입니다. HyperCLOVA X와 같은 LLM 서비스는 입/출력에 사용된 토큰 수를 기반으로 과금됩니다.
HyperCLOVA X의 안전한 AI(Safety)를 위해 CLOVA Studio에서는 **유해 콘텐츠 필터링(Content Filtering)**을 기본 제공합니다. 이는 혐오 표현이나 위험한 답변을 사전에 차단하기 위한 시스템입니다. 콘텐츠 필터는 욕설 및 비속어, 개인정보 유출 위험, 자해 및 폭력적 내용, 정치적 편향성 및 차별 등 다양한 유해 항목을 실시간으로 감지합니다.
AI 모델이 사실이 아닌 정보를 마치 진실인 것처럼 그럴싸하게 답변하는 현상을 **할루시네이션(Hallucination)**이라고 합니다. 이는 모델이 학습 데이터에 없는 거짓 정보를 생성하는 현상으로, RAG(Retrieval-Augmented Generation) 등을 통해 완화할 수 있습니다.
Chapter 7. HyperCLOVA X의 특별함
HyperCLOVA X가 한국어 처리에 강점을 가지는 근본적인 이유는 한국어와 한국 문화 관련 데이터를 대규모로 학습했기 때문입니다. 방대한 양의 한국어 뉴스, 지식iN 등 국내 데이터를 학습하여 한국적 맥락에 최적화되어 있습니다. 이는 타 글로벌 모델 대비 한국어 비즈니스 환경에서 우위를 점하는 핵심 이유입니다.
HyperCLOVA X는 한국어 특화 토크나이저를 활용하여 해외 LLM 대비 최고 2배까지 빠르고 저렴합니다. 또한 이미지와 음성 정보를 한국어로 추론 가능한 네이티브 멀티모달 기능을 갖추고 있어, 텍스트뿐만 아니라 다양한 형태의 입력을 처리할 수 있습니다.
HyperCLOVA X 모델 중 HCX-DASH는 응답 속도가 빠르고 비용 효율적이어서 대규모 단순 반복 작업에 최적화된 경량화 모델입니다. 속도와 비용에 최적화되어 높은 효율성을 필요로 하는 서비스에 적합합니다.
Chapter 8. 고급 기능과 활용
**시스템 메시지(System Message)**는 AI의 역할(Role)을 정의하고, 답변의 말투(Tone & Manner)를 지정하며, 대화의 규칙이나 제약사항을 설정하는 배경 설정 정보입니다. 사용자가 입력하는 구체적인 질문은 'User' 메시지에 해당합니다.
**페르소나 설정(Persona Setting)**은 모델에게 "너는 20년 경력의 전문 변호사야"라고 역할을 부여하는 기법입니다. 이는 모델에게 특정 정체성을 부여하여 답변의 전문성과 톤앤매너를 제어하는 기법입니다.
HyperCLOVA X 모델이 가진 지식을 넘어, 사내 문서 등을 검색하여 답변하도록 하는 RAG 구성 시 필수적인 API는 임베딩(Embedding) API입니다. 문서를 벡터화하여 검색하기 위해서는 텍스트를 숫자로 변환하는 임베딩 API가 반드시 필요합니다.
Skill Trainer를 사용하여 외부 API를 연동할 때는 OpenAPI Spec(Swagger) 형식으로 API의 명세(기능 설명, 파라미터 등)를 작성합니다. Skill 등록 시 표준 규격인 OpenAPI Specification을 사용하여 API의 구조를 정의합니다.
모델이 복잡한 추론을 수행할 때, 답만 요구하는 것이 아니라 단계별 사고 과정을 출력하도록 유도하여 정확도를 높이는 기법을 **Chain of Thought(CoT)**라고 합니다. "단계별로 생각해서 풀이해 줘"라는 지시어는 모델이 논리적인 단계를 거쳐 결론에 도달하게 함으로써 복잡한 문제 해결 능력을 향상시킵니다.
프롬프트 작성 시 아무런 예시를 주지 않고 지시문만으로 결과물을 얻는 방식은 Zero-shot이며, 한 개의 예시를 제공하는 방식은 One-shot, 여러 개의 예시를 제공하는 방식은 Few-shot입니다.
Chapter 9. CLOVA Studio 서비스 유형
CLOVA Studio는 세 가지 주요 서비스 유형을 제공합니다.
CLOVA Studio Basic은 토큰 사용량에 따라 요금을 지불하며 신규 기능을 빠르게 적용받을 수 있는 일반 사용자 대상 서비스입니다.
CLOVA Studio Exclusive는 Advanced Tuning 전용 인프라를 제공하며 전용 리소스가 할당되는 기업 고객 대상 서비스입니다.
Neurocloud for HyperCLOVA X는 고객사 IDC에 하이브리드 클라우드 형태로 제공하는 완전관리형 서비스로, 데이터 보안이 극도로 중요한 기업을 위해 네이버 클라우드의 인프라를 고객사 내부에 직접 구축하여 데이터 유출 우려 없이 HyperCLOVA X를 사용할 수 있게 해주는 전용 서비스입니다.
CLOVA Studio의 데이터 보안 정책은 매우 엄격합니다. 사용자가 입력한 프롬프트는 모델의 재학습 데이터로 자동 사용되지 않으며, 기업용 서비스의 경우 전용 인프라를 통한 격리된 environment 제공이 가능합니다. API를 통해 전송되는 데이터는 암호화되어 처리되고, 튜닝에 사용된 데이터는 해당 프로젝트 내에서만 접근 가능합니다.
발급받은 API Key가 외부에 노출되었을 때 가장 먼저 취해야 할 조치는 API Key 재발급 및 기존 키 무효화입니다. API Key 노출 시 즉시 재발급(Rotate) 기능을 통해 기존 키의 권한을 박탈하여 오남용을 막아야 합니다.

📘 Part 2: CLOVA Chatbot - 대화형 AI의 구현
Chapter 1. CLOVA Chatbot 기본 구조
CLOVA Chatbot 서비스에서 사용자의 질문 의도를 파악하고 분류하기 위해 설정하는 가장 기본 단위는 **인텐트(Intent)**입니다. 인텐트는 사용자가 챗봇에게 질문하는 목적을 정의하는 핵심 요소입니다.
사용자의 문장 속에서 특정 정보(날짜, 장소, 상품명 등)를 추출하기 위해 사용하는 사전 형태의 데이터는 **엔티티(Entity)**입니다. 엔티티는 문장 내에서 의미 있는 핵심 단어를 추출하고 분류하기 위해 사용됩니다.
CLOVA Chatbot 도메인을 처음 생성할 때는 도메인 이름과 **서비스 언어(한국어, 영어 등)**를 필수로 설정해야 합니다. 메신저 연동은 생성 이후에 진행할 수 있습니다.
Chapter 2. 다국어 지원과 답변 형태
CLOVA Chatbot은 한국어, 영어, 일본어, 중국어, 태국어, 인도네시아어 총 6개 언어를 지원합니다. 각 언어별 특징에 맞는 최적화된 학습 알고리즘을 제공합니다.
챗봇에서 지원하는 답변 형태는 다양합니다. 텍스트 답변, 이미지 답변, 카드/리스트 답변, 퀵 리플라이(Quick Reply) 버튼 등이 있습니다. 이를 통틀어 답변 컴포지트라고 하며, CLOVA Chatbot은 5가지 형식을 제공합니다.
답변 하단에 생성되어 사용자가 클릭 한 번으로 다음 질문을 할 수 있게 돕는 기능은 **퀵 리플라이(Quick Reply)**입니다. 버튼 형태로 나타나 사용자의 다음 행동을 유도하는 편의 기능입니다.
이미지와 버튼이 가로로 나열되는 형태의 메시지 타입은 캐러셀(Carousel) / 카드 타입입니다. 여러 개의 카드를 가로로 슬라이드하며 보여주는 형태로, 상품 목록이나 메뉴 추천에 자주 사용됩니다.
Chapter 3. 신뢰도와 빌드 프로세스
챗봇이 사용자의 질문에 답변을 결정하는 '신뢰도 점수'의 기준이 되는 임계치 설정을 Confidence Score Threshold라고 합니다. 모델이 분석한 결과가 설정값보다 높을 때만 답변을 출력하도록 제한하는 기준입니다.
챗봇 대화 모델을 학습시킨 후, 실제 서비스에 적용하기 위해 엔진에 데이터를 반영하는 과정은 **빌드(Build)**입니다. 변경된 대화 데이터가 실제 챗봇 응답에 반영되려면 반드시 빌드 과정을 거쳐야 합니다.
챗봇의 인식률을 높이기 위해 '유사 질문'을 등록할 때는 의미가 같은 다양한 형태의 문장을 등록하고, 오타가 포함된 예상 발화도 등록하며, 최소 5~10개 이상의 데이터를 입력하는 것이 좋습니다. 다양한 형태와 적정량의 데이터를 입력해야 모델의 학습 효과가 극대화됩니다.
Chapter 4. 시스템 엔티티와 슬롯 필링
CLOVA Chatbot에서 기본으로 제공하는 '시스템 엔티티'로는 @sys.date(날짜), @sys.time(시간), @sys.number(숫자) 등이 있습니다. 시스템 엔티티는 네이버에서 미리 정의해 둔 데이터 타입이며, 사용자 이름 등은 커스텀 엔티티로 구성해야 합니다.
사용자가 "피자 주문해줘"라고 했을 때, "어떤 피자를 주문하시겠어요?"라고 되묻는 것과 같이 필수 정보를 채우기 위해 사용하는 기능은 **슬롯 필링(Slot Filling)**입니다. 이는 의도를 완료하기 위해 필요한 필수 파라미터(엔티티)가 누락되었을 때 되물어서 채우는 과정입니다.
슬롯 필링 과정에서 사용자가 엉뚱한 답변을 계속할 때, 챗봇이 되묻기를 중단하고 실패 처리하는 횟수를 지정하는 설정은 **최대 재시도 횟수(Max Retries)**입니다. 무한 루프를 방지하기 위해 되묻기 횟수를 제한하고, 초과 시 기본 실패 메시지를 출력하거나 상담원을 연결합니다.
Chapter 5. 외부 연동과 액션 메소드
외부 서버와 연동하여 실시간 재고 확인이나 날씨 정보를 가져와 답변에 포함시키고 싶을 때 사용하는 기능은 **액션 메소드(Action Method)**입니다. 외부 URL(API)을 호출하여 동적인 데이터를 답변에 반영할 때 사용합니다.
CLOVA Chatbot을 카카오톡, 라인 등 외부 채널과 연동하기 위해 통신 중간 역할을 하는 네이버 클라우드의 서비스는 API Gateway입니다. 챗봇과 외부 메신저 간의 안정적인 데이터 전송을 위해 API Gateway 연동이 필요합니다.
액션 메소드 호출 시 챗봇 서버가 외부 API로 전달하는 기본 정보(Payload)에는 사용자 식별값(User ID), 매칭된 인텐트 이름, 채워진 슬롯(엔티티) 정보 등이 포함됩니다. 챗봇은 분석된 의도(Intent)와 파라미터(Entities), 사용자 정보 등을 JSON 형태로 외부 서버에 전달합니다.
CLOVA Chatbot의 액션 메소드 연동 방식으로는 **HTTP 통신(JSON 형식)**과 Cloud Functions 연동이 있습니다. 액션 메소드는 보안 및 관리 효율을 위해 HTTP API 호출이나 Cloud Functions를 통해 외부와 통신합니다.
Chapter 6. 통계와 관리
챗봇의 '통계' 메뉴에서 분석할 수 있는 주요 지표로는 총 호출 건수(대화 수), 인텐트별 매칭 순위, 미응답 질문 비율, 사용자 유입 채널별 비중 등이 있습니다. 통계 메뉴는 챗봇의 성능과 사용자 이용 행태를 분석하기 위한 다양한 데이터를 제공합니다.
챗봇이 이해하지 못한 질문(실패 질문)을 확인하고 이를 학습 데이터로 추가할 수 있는 메뉴는 미응답 질문 관리입니다. 미응답 질문 관리 메뉴에서 사용자의 실제 발화 중 답변을 찾지 못한 질문들을 확인하고 개선할 수 있습니다.
CLOVA Chatbot 서비스의 요금 산정 시 가장 큰 비중을 차지하는 기준은 **월간 대화 요청 건수(API Call)**입니다. 실제 사용자가 챗봇과 나눈 대화 호출 수를 기준으로 과금이 진행됩니다.
Domain Group의 API 호출은 Transaction 기준 2회 호출로 카운트됩니다. 이는 비용 관리 시 중요하게 고려해야 할 사항입니다.
